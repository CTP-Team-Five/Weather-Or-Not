# ADR 0000: Use ADRs

## Status
Accepted

## Context
Keeping an external track of softwares and hardwares needed to run our app as well as inform others on what is to be expected if using WeatherOrNot

## Decision
We will use **ADRs** for this project.

## Benefits
- Clear visualization of the process of decesion making.
- Ability to link ADRs that are related together.

## Consequences
- Upkeep my be necessary if things change.
